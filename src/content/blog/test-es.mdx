# Título

**Si conoces HTML, ya sabes lo suficiente para escribir tu primer componente de Astro.**

La sintaxis del componente Astro es un superconjunto de HTML. La sintaxis fue [diseñada para sentirse familiar para cualquiera con experiencia escribiendo HTML o JSX](#diferencias-entre-astro-y-jsx), y añade soporte para incluir componentes y expresiones de JavaScript.

## Expresiones tipo JSX

Puedes definir variables locales de JavaScript dentro del script del componente en la materia frontal entre los dos delimitadores de código (`---`) de un componente Astro. ¡Luego puedes inyectar estas variables en la plantilla HTML del componente usando expresiones tipo JSX!

nota\[dinámico vs reactivo]
Usando este enfoque, puedes incluir valores **dinámicos** que se calculan en la materia frontal. Pero una vez incluidos, estos valores no son **reactivos** y nunca cambiarán. Los componentes de Astro son plantillas que solo se ejecutan una vez, durante el paso de renderizado.

Ver más abajo para más ejemplos de [diferencias entre Astro y JSX](#diferencias-entre-astro-y-jsx).

### Variables

Las variables locales se pueden agregar al HTML usando la sintaxis de llaves:

```html
---
const nombre = 'Astro'
---

<div>
	<h1>¡Hola {nombre}!</h1>
	<!-- Muestra <h1>¡Hola Astro!</h1> -->
</div>
```

### Atributos Dinámicos

Las variables locales se pueden usar en llaves para pasar valores de atributos tanto a elementos HTML como a componentes:

```astro
---
const nombre = 'Astro'
---

<h1 class={nombre}>Las expresiones de atributos son compatibles</h1>

<MyComponent templateLiteralNameAttribute={`MiNombreEs${nombre}`} />
```

advertencia
Los atributos HTML se convertirán en cadenas, por lo que no es posible pasar funciones y objetos a elementos HTML.
Por ejemplo, no puedes asignar un manejador de eventos a un elemento HTML en un componente Astro:

```astro
---
// no-hagas-esto.astro
function handleClick() {
	console.log('¡botón clickeado!')
}
---

<!-- ❌ ¡Esto no funciona! ❌ -->
<button onClick={handleClick}>Nada sucederá cuando me cliques</button>
```

En su lugar, usa un script del lado del cliente para agregar el manejador de eventos, como lo harías en JavaScript vainilla:

```astro
---
// haz-esto-en-cambio.astro
---

<button id='button'>Clickeame</button>
<script>
	function handleClick() {
		console.log('¡botón clickeado!')
	}
	document.getElementById('button').addEventListener('click', handleClick)
</script>
```

## Expresiones

### HTML Dinámico

Las variables locales se pueden usar en funciones tipo JSX para producir elementos HTML generados dinámicamente:

```astro
---
const items = ['Perro', 'Gato', 'Ornitorrinco']
---

<ul>
	{items.map((item) => <li>{item}</li>)}
</ul>
```

Astro puede mostrar HTML condicionalmente usando operadores lógicos JSX y expresiones ternarias.

```astro
---
const visible = true
---

{visible && <p>¡Muéstrame!</p>}

{visible ? <p>¡Muéstrame!</p> : <p>O muéstrame esto!</p>}
```

### Etiquetas Dinámicas

También puedes usar etiquetas dinámicas estableciendo una variable a un nombre de etiqueta HTML o una importación de componente:

```astro
---
import MyComponent from './MyComponent.astro'
const Elemento = 'div'
const Componente = MyComponent
---

<Elemento>¡Hola!</Elemento>
<!-- se renderiza como <div>¡Hola!</div> -->
<Componente />
<!-- se renderiza como <MyComponent /> -->
```

Cuando usas etiquetas dinámicas:

- **Los nombres de las variables deben estar en mayúsculas.** Por ejemplo, usa `Elemento`, no `elemento`. De lo contrario, Astro intentará renderizar tu nombre de variable como una etiqueta HTML literal.
- **Las directivas de hidratación no son compatibles.** Cuando usas las [directivas de hidratación `cliente:*`](/es/conceptos-principales/componentes-de-marco/#hidratando-componentes-interactivos), Astro necesita saber qué componentes empaquetar para la producción, y el patrón de etiqueta dinámica impide que esto funcione.

### Fragmentos

Astro soporta usar tanto `<Fragment> </Fragment>` como el atajo `<> </>`.

Los fragmentos pueden ser útiles para evitar elementos envolventes cuando agregas [`directivas set:*`](/es/referencia/directivas-referencia/#sethtml), como en el siguiente ejemplo:

```astro
---
const htmlString = '<p>Contenido HTML crudo</p>'
---

<Fragment set:html={htmlString} />
```

### Diferencias entre Astro y JSX

La sintaxis del componente Astro es un superconjunto de HTML. Fue diseñada para sentirse familiar para cualquiera con experiencia en HTML o JSX, pero hay un par de diferencias clave entre los archivos `.astro` y JSX.

#### Atributos

En Astro, usas el formato estándar `kebab-case` para todos los atributos HTML en lugar del `camelCase` utilizado en JSX. Esto también funciona para `class`, que no es compatible con React.

```jsx
<div className="box" dataValue="3" />
<div class="box" data-value="3" />
```

#### Múltiples Elementos

Una plantilla de componente Astro puede renderizar múltiples elementos sin necesidad de envolver todo en un solo `<div>` o `<>`, a diferencia de JavaScript o JSX.

```astro
---
// Plantilla con múltiples elementos
---

<p>No necesitas envolver elementos en un solo elemento contenedor.</p>
<p>Astro soporta múltiples elementos raíz en una plantilla.</p>
```

#### Comentarios

En Astro, puedes usar comentarios HTML estándar o comentarios al estilo JavaScript.

```astro
---

---

<!-- La sintaxis de comentarios HTML es válida en archivos .astro -->{/* La sintaxis de comentarios JS también es válida */}
```
